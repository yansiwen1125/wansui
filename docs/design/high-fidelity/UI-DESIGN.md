# 万岁 V1.0 UI 设计规范

## 1. 设计方向

关键词：**安静、温暖、清楚、轻量**。

界面使用偏纸张感的暖白背景与深墨色文字，四种任务色只用于任务识别和完成反馈，不大面积铺色。整体避免传统效率工具的冰冷感，也不加入与 V1 功能无关的装饰。

## 2. 设计基准

- 基准画布：`390 × 844 CSS px`，仅作为设计参考，不是前端固定宽度
- 页面布局：移动端优先、宽度自适应
- 应用主体：`width: 100%`，建议 `max-width: 480px` 并水平居中
- 内容左右边距：`24px`
- 底部导航高度：`72px + safe-area`
- 最小点击区域：`44 × 44px`
- 圆角体系：`10 / 16 / 22px`
- 间距体系：`4 / 8 / 12 / 16 / 24 / 32 / 40px`

响应式规则：

- 页面在 `320px–480px` 视口范围内连续自适应。
- 页面左右边距可由 `24px` 平滑缩小至 `16px`。
- 任务名称保持单行，超出时缩小到 `16px`，不换行。
- 月历和周摘要等比分配可用宽度。
- 任务记录的三个月区域保持横向滑动，不压缩圆点为椭圆。

## 5. 核心组件

### 日期导航

- 日期文字居中，左右箭头视觉尺寸 `20px`、点击区 `44px`。
- 可用箭头为主文字色，不可用为弱文字色。
- 使用浅灰色圆角按钮背景，不显示描边。

### 周田字格

- 单个小方块 `10px`，间距 `3px`，圆角 `3px`。
- 选中日期使用 `2px` 黑色描边。
- 今天未选中时使用 `1.5px` 中灰描边。
- 普通过去日期无外框。

### 任务卡片

- 高度 `68px`，圆角 `18px`，白色背景。
- 左侧任务色块 `12 × 36px`，圆角 `6px`。
- 未完成卡片使用浅灰描边；已完成使用对应任务色描边和极浅任务色背景。
- 勾选按钮视觉尺寸 `28px`，点击区 `44px`。

### 月历日期单元

- 每列等宽，日期数字位于九宫格上方。
- 今天使用黑色描边。
- 当天完成率达到 50% 时使用低饱和米棕色 `#EFEAE2` 背景。
- 九宫格内未打卡和未占用位置统一使用 `#D8D6D0`。
- 非本月日期不另换灰色，只将整个日期单元降低透明度。

### 底部导航

- 白色半透明背景，上方 `1px` 分割线。
- 选中项使用主文字色和实心图形，未选中项使用弱文字色。
- 图标与文字整体点击区域不小于 `64 × 52px`。

## 6. 状态反馈

- **加载中**：内容区域显示轻量环形进度，不使用整页遮罩。
- **离线只读**：顶部出现浅米色状态条「离线，只能查看」。
- **保存失败**：底部浮层提示「保存失败，请稍后重试」，原勾选状态不变化。
- **用户名错误**：输入框描边变为棕红色，下方显示「用户名不正确」。
- **点击反馈**：卡片按下缩放至 `0.98`，时长 `120ms`；勾选完成使用 `160ms` 淡入。
