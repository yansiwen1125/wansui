# 万岁 V1.0 设计需求对齐

## 角色

**产品设计师：小满**

负责登录页、首页、月历、任务记录页的交互与视觉规范。本期只实现固定用户名登录、打卡和记录功能，不增加注册、密码、验证码、任务管理、提醒或成就功能。

## 已对齐需求

- 系统预制 4 个固定任务：拉屎、早睡、运动、饮食健康。
- 固定顺序与颜色：
  - 拉屎：棕色 `#A9795B`
  - 早睡：深蓝色 `#315F8A`
  - 运动：淡蓝色 `#82BDE3`
  - 饮食健康：绿色 `#7FB77E`
- 首页可切换 `2026-06-10` 起（含当天）至今天之间的日期，标题固定为「我做了…」。
- 首页周摘要按周日至周六排列，田字格的 4 个位置固定对应 4 个任务。
- 月历以九宫格展示任务状态；V1 只占用前 4 格，其余 5 格为灰色空位。
- 任务记录按任务纵向排列，每个任务横向展示最近 3 个月。
- 任务记录不能滑动到未来月份。
- 月历和任务页使用统一的文字 Tab。
- V1 是可添加到 iPhone 主屏幕的网页 App。
- 增加极简用户名登录页，只包含用户名输入框和进入按钮。
- 需要设计用户名错误、加载中、离线只读和保存失败状态。

## 已确定的视觉规范

### 1. 颜色

完整色板已写入 [UI 设计规范](../design/high-fidelity/UI-DESIGN.md)。页面使用暖白背景、深墨色文字；九宫格未完成状态统一使用一种灰色，完成率背景使用极浅棕色。

### 2. 点击区域

箭头、勾选框和日期单元的视觉尺寸可以较小，但网页实际可点击区域统一不小于 `44×44 CSS px`。

### 3. 响应式适配

- 高保真稿的 `390px` 宽度仅作为视觉基准，前端页面必须自适应手机宽度。
- `320px–480px` 视口范围内页面边距从 `16px` 至 `24px` 平滑变化。
- 任务名称保持单行，必要时字号从 `17px` 缩小至 `16px`，不换行。
- 三个月圆点区域保持横向滑动，不通过非等比缩放挤压圆点。

### 4. 网络状态

使用小型状态文字表达“离线，只能查看”；离线点击勾选时再提示「网络不可用，暂时不能打卡」。

## 建议改进

- 制作一份组件状态表：默认、已完成、未完成、今天、选中日期、未来日期、非本月空位。
- 日期导航在到达固定生效日期 `2026-06-10` 和今天时，分别禁用对应方向箭头。
- 月历完成率背景使用浅棕色，与四种任务色区分。
- 网格不能只依赖颜色表达“今天”，继续保留黑色描边。
- 所有灰色至少分为两级：本月未打卡、非本月或未占用；后者应更浅。
- 高保真稿需要统一页面左右边距、Tab 基线、分割线位置和底部导航安全区。
- 补充登录页、用户名错误、离线只读和保存失败状态。
- 动效保持克制，只需为勾选状态变化提供轻微即时反馈。
